# 用户表 (sys_user)

对应“用户新增/编辑”页面。

| 字段名               | 数据类型     | 必填 | 说明                                           |
| -------------------- | ------------ | ---- | ---------------------------------------------- |
| `id`                 | BIGINT       | 是   | 主键，自增 ID。                                |
| `username`           | VARCHAR(50)  | 是   | **用户名（工号）**，登录账号，需唯一。         |
| `password`           | VARCHAR(100) | 是   | **密码**，加密存储（如 BCrypt）。              |
| `real_name`          | VARCHAR(50)  | 是   | **姓名**。                                     |
| `gender`             | TINYINT      | 否   | **性别**：0-未知，1-男，2-女。                 |
| `dept_id`            | BIGINT       | 是   | **所属部门 ID**，关联 `sys_dept.id`。          |
| `job_title`          | VARCHAR(50)  | 否   | **职位**（如：经理、专员）。                   |
| `email`              | VARCHAR(100) | 是   | **邮箱**。                                     |
| `phone`              | VARCHAR(20)  | 是   | **手机号**。                                   |
| `professional_level` | VARCHAR(20)  | 否   | **专业等级**（字典值或关联表 ID）。            |
| `business_level`     | VARCHAR(20)  | 否   | **商务等级**（字典值或关联表 ID）。            |
| `employment_status`  | TINYINT      | 是   | **离职在职状态**：0-离职，1-在职（对应开关）。 |
| `account_status`     | TINYINT      | 是   | **账号状态**：0-禁用，1-启用（对应开关）。     |
| `created_at`         | TIMESTAMP    | 是   | 创建时间。                                     |
| `updated_at`         | TIMESTAMP    | 是   | 更新时间。                                     |

# 部门表 (sys_dept)

对应“编辑部门”弹窗及左侧菜单中的“部门管理”。

| 字段名        | 数据类型     | 必填 | 说明                                                      |
| ------------- | ------------ | ---- | --------------------------------------------------------- |
| `id`          | BIGINT       | 是   | 主键，自增 ID。                                           |
| `parent_id`   | BIGINT       | 是   | **父部门 ID**，顶级部门为 0，用于构建树形结构。           |
| `dept_name`   | VARCHAR(50)  | 是   | **部门名称**（如：测试部）。                              |
| `dept_code`   | VARCHAR(30)  | 是   | **部门编码**（如：198786），通常需唯一。                  |
| `leader_id`   | BIGINT       | 否   | **负责人 ID**，关联 `sys_user.id`（点击放大镜搜索用户）。 |
| `description` | VARCHAR(255) | 否   | **部门描述**。                                            |
| `remark`      | TEXT         | 否   | **备注**。                                                |
| `status`      | TINYINT      | 是   | 状态：0-停用，1-正常。                                    |

# 角色表 (sys_role)

对应“角色管理”列表页。

# 菜单/权限表 (sys_menu)

对应“权限设置”中的树形勾选框。这张表既存左侧导航菜单，也存页面上的按钮权限（增删改查）。

| 字段名       | 数据类型     | 必填 | 说明                                                   |
| ------------ | ------------ | ---- | ------------------------------------------------------ |
| `id`         | BIGINT       | 是   | 主键，自增 ID。                                        |
| `parent_id`  | BIGINT       | 是   | **父菜单 ID**，顶级菜单为 0。                          |
| `menu_name`  | VARCHAR(50)  | 是   | **菜单名称**（如：客户主菜单、我的客户）。             |
| `menu_type`  | CHAR(1)      | 是   | **菜单类型**：M-目录，C-菜单，F-按钮（权限点）。       |
| `perms`      | VARCHAR(100) | 否   | **权限标识**（如 `system:user:add`），仅按钮类型需要。 |
| `path`       | VARCHAR(200) | 否   | 路由地址（前端跳转路径）。                             |
| `icon`       | VARCHAR(100) | 否   | 菜单图标。                                             |
| `sort_order` | INT          | 否   | 显示排序。                                             |

### `9.sys_economy_category` (国民经济行业分类表)

| 字段名          | 类型         | 必填 | 说明                                                         |
| --------------- | ------------ | ---- | ------------------------------------------------------------ |
| `id`            | BIGINT       | ✅    | 主键 ID                                                      |
| `parent_id`     | BIGINT       | ✅    | 父级 ID。顶级（门类）为 0。用于构建树形结构。                |
| `category_code` | VARCHAR(10)  | ✅    | 行业代码。例如：`B`, `07`, `072`, `0721`。这是核心索引字段。 |
| `category_name` | VARCHAR(100) | ✅    | 行业名称。例如：`采矿业`, `石油和天然气开采业`。             |
| `level`         | TINYINT      | ✅    | 层级深度。 1: 门类 (A-T) 2: 大类 (两位数字) 3: 中类 (三位数字) 4: 小类 (四位数字) |
| `full_path`     | VARCHAR(255) | ❌    | 全路径名称 (可选优化)。例如："B 采矿业 > 07 石油... > 0721 陆地..." *作用：避免递归查询，加速列表展示。* |
| `sort_order`    | INT          | ❌    | 排序权重。通常按代码顺序排列即可。                           |
| `status`        | TINYINT      | ✅    | 状态：1-启用，0-停用。                                       |

~~~sql
CREATE TABLE `sys_economy_category` (
  `id` bigint(20) NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `parent_id` bigint(20) NOT NULL DEFAULT '0' COMMENT '父级ID，顶级为0',
  `category_code` varchar(10) NOT NULL COMMENT '行业代码 (如: B, 07, 072, 0721)',
  `category_name` varchar(100) NOT NULL COMMENT '行业名称 (如: 陆地天然气开采)',
  `level` tinyint(4) NOT NULL COMMENT '层级: 1-门类, 2-大类, 3-中类, 4-小类',
  `full_path` varchar(255) DEFAULT NULL COMMENT '层级全路径名称 (冗余字段，便于搜索和展示)',
  `sort_order` int(11) DEFAULT '0' COMMENT '排序',
  `status` tinyint(4) DEFAULT '1' COMMENT '状态: 1-有效, 0-无效',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_code` (`category_code`), -- 代码必须唯一
  KEY `idx_parent_id` (`parent_id`) -- 优化级联查询性能
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='国民经济行业分类表';
~~~
