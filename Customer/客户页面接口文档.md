# 客户页面接口文档

## 1. 通用约定

### 1.1 Base URL

- **本地开发**：`http://127.0.0.1:8000`

### 1.2 鉴权方式

默认请求头约定:

- `Bearer <token>`

### 1.3 统一响应格式

```json
{
    "success": true,
    "message": "操作成功",
    "data": {}
}
```

字段说明:

- `success`: 业务状态码
- `message`: 响应提示信息
- `data`: 返回业务数据，可能为对象、数组或`null`

### 1.4 分页响应格式

分页接口的`meta`统一为:

```json
{
   "current_page": 1,
   "per_page": 15,
   "total": 100,
   "last_page": 7
}
```

- `total`: 总记录数
- `current_page`: 当前页码
- `per_page`: 每页条数
- `last_page`: 最后一页条数

# 用户管理模块 (sys\_user)--LX

对应界面：**图 1（新增/编辑用户弹窗）**

#### 1.1 新增用户

- **接口地址**：`POST /api/v1/users`
- **描述**：创建一个新的系统用户账号。
- **请求体 (Request Body)**：

```json
{
  "username": "zhangsan",        // [必填] 用户名/工号
  "password": "123456",          // [必填] 密码（前端可明文传输，后端加密存储）
  "realName": "张三",            // [必填] 姓名
  "gender": 1,                   // [选填] 性别：1-男，2-女
  "deptId": 101,                 // [必填] 所属部门ID
  "jobTitle": "专利代理师",       // [选填] 职位
  "email": "zhangsan@example.com", // [必填] 邮箱
  "phone": "13800138000",        // [必填] 手机号
  "professionalLevel": "LEVEL_A",// [选填] 专业等级（字典值）
  "businessLevel": "LEVEL_B",    // [选填] 商务等级（字典值）
  "status": 1,                   // [必填] 账号状态：1-启用，0-禁用
  "employmentStatus": 1          // [必填] 在职状态：1-在职，0-离职
}
```

#### 1.2 修改用户

- **接口地址**：`PUT /api/v1/users/{id}`
- **描述**：更新现有用户信息。
- **路径参数**：`id` (用户 ID)
- **请求体**：同“新增用户”，`password` 字段若为空则不修改密码。

#### 1.3 获取用户详情

- **接口地址**：`GET /api/v1/users/{id}`
- **描述**：回显编辑页面的数据。
- **响应示例 (Data)**：

```json
{
  "id": 1001,
  "username": "zhangsan",
  "realName": "张三",
  "gender": 1,
  "deptId": 101,
  "deptName": "专利部", // 建议后端直接返回名称，方便前端回显
  "jobTitle": "专利代理师",
  "email": "zhangsan@example.com",
  "phone": "13800138000",
  "professionalLevel": "LEVEL_A",
  "businessLevel": "LEVEL_B",
  "status": 1,
  "employmentStatus": 1
}
```

# 角色管理模块 (sys\_role)--LX

对应界面：**图 2（角色列表）、图 3（权限设置树）**

#### 2.1 创建角色

- **接口地址**：`POST /api/v1/roles`
- **描述**：新增一个系统角色。
- **请求体**：

```json
{
  "roleName": "客服人员",
  "roleKey": "customer_service",
  "description": "负责客户咨询与问题处理",
  "status": 1,
  "sortOrder": 1
}
```

- **响应示例 (Data)**：

```json
{
  "id": 1,
  "roleName": "客服人员",
  "roleKey": "customer_service",
  "description": "负责客户咨询与问题处理",
  "status": 1,
  "sortOrder": 1,
  "createTime": "2026-06-26T15:30:00"
}
```

#### 2.2 角色分页列表

- **接口地址**：`GET /api/v1/roles`
- **描述**：获取角色表格数据。
- **查询参数**：

| 参数名        | 类型      | 必填 | 说明         |
| ---------- | ------- | -- | ---------- |
| `pageNum`  | Integer | 否  | 页码，默认 1    |
| `pageSize` | Integer | 否  | 每页条数，默认 10 |
| `roleName` | String  | 否  | 角色名称模糊搜索   |

- **响应示例 (Data)**：

```json
{
  "total": 100,
  "rows": [
    {
      "id": 7,
      "roleName": "客服人员",
      "roleKey": "customer_service",
      "description": "用于管理系统用户1",
      "status": 1,
      "createTime": "2026-01-29T08:11:00"
    }
  ]
}
```

#### 2.3 获取角色详情

- **接口地址**：`GET /api/v1/roles/{id}`
- **描述**：获取单个角色的详细信息。
- **路径参数**：`id` (角色 ID)
- **响应示例 (Data)**：

```json
{
  "id": 1,
  "roleName": "客服人员",
  "roleKey": "customer_service",
  "description": "负责客户咨询与问题处理",
  "status": 1,
  "sortOrder": 1,
  "createTime": "2026-06-26T15:30:00"
}
```

#### 2.4 修改角色

- **接口地址**：`PUT /api/v1/roles/{id}`
- **描述**：更新角色名称、描述、状态等信息。
- **路径参数**：`id` (角色 ID)
- **请求体**：同 2.1 创建角色
- **响应示例 (Data)**：同 2.3 获取角色详情

#### 2.5 删除角色

- **接口地址**：`DELETE /api/v1/roles/{id}`
- **描述**：删除指定角色。
- **路径参数**：`id` (角色 ID)
- **响应示例 (Data)**：`null`

#### 2.6 分配/保存权限

- **接口地址**：`PUT /api/v1/roles/{id}/menus`
- **描述**：对应图 3 的“保存”操作，将勾选的菜单 ID 绑定给角色。
- **路径参数**：`id` (角色 ID)
- **请求体**：

```json
{
  "menuIds": [1, 2, 5, 10, 12] // 所有被勾选的菜单/按钮ID集合
}
```

#### 2.8 创建菜单

- **接口地址**：`POST /api/v1/menus`
- **描述**：新增一个系统菜单/权限节点。
- **请求体**：

```json
{
  "parentId": 0,              // [选填] 父菜单ID，0为顶级目录
  "menuName": "客户主菜单",    // [必填] 菜单名称
  "menuType": "M",            // [必填] M-目录，C-菜单，F-按钮
  "perms": "customer:view",   // [选填] 权限标识
  "path": "/customer",        // [选填] 路由地址
  "icon": "el-icon-user",     // [选填] 菜单图标
  "sortOrder": 1              // [选填] 显示排序
}
```

- **响应示例 (Data)**：

```json
{
  "id": 1,
  "parentId": 0,
  "menuName": "客户主菜单",
  "menuType": "M",
  "perms": "customer:view",
  "path": "/customer",
  "icon": "el-icon-user",
  "sortOrder": 1
}
```

#### 2.9 获取菜单树形结构

- **接口地址**：`GET /api/v1/menus/tree`
- **描述**：获取所有菜单的树形结构，用于权限设置弹窗左侧展示可勾选的菜单列表。
- **响应示例 (Data)**：

```json
[
  {
    "id": 1,
    "label": "客户主菜单",
    "menuType": "M",
    "children": [
      {
        "id": 2,
        "label": "客户列表",
        "menuType": "C"
      },
      {
        "id": 3,
        "label": "我的客户",
        "menuType": "C"
      }
    ]
  }
]
```

# 部门管理模块 (sys\_dept)--LX

对应界面：**图 4（编辑部门弹窗）**

#### 3.1 新增/编辑部门

- **接口地址**：`POST /api/v1/depts` (新增) 或 `PUT /api/v1/depts/{id}` (编辑)
- **描述**：保存部门信息。
- **请求体**：

```json
{
  "parentId": 0,           // [必填] 父部门ID，0代表顶级部门
  "deptName": "测试bx",     // [必填] 部门名称
  "deptCode": "198786",     // [必填] 部门编码
  "leaderName": "李鑫",     // [选填] 负责人姓名（或者传 leaderId）
  "description": "这是一个测试部门", // [选填] 部门描述
  "remark": "备注信息"      // [选填] 备注
}
```

#### 3.2 获取部门树形结构

- **接口地址**：`GET /api/v1/depts/tree`
- **描述**：通常用于左侧导航栏展示，或者在“选择所属部门”时的下拉树选择器。
- 子部门中的parentid传入数据库里面父级所对应的id
- **响应示例 (Data)**：

```json
[
  {
    "id": 100,
    "label": "总公司",
    "children": [
      {
        "id": 101,
        "label": "研发部",
        "children": []
      },
      {
        "id": 102,
        "label": "市场部",
        "children": []
      }
    ]
  }
]
```
